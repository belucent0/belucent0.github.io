# Draft Templates

Use these files as copy-and-paste starters when writing in `github.dev` or a local editor.

Typical workflow:

1. Copy one template from this folder.
2. Create a new post under `_posts/`.
3. Name it with the Jekyll convention, for example:

   ```text
   _posts/2026-06-22-my-engineering-note.md
   ```

4. Replace the title, date, categories, tags, and body.
5. Commit the new file to `main`; GitHub Actions will publish the site.

Suggested categories:

- `engineering-notes`: software engineering lessons, production debugging, system design notes.
- `project-logs`: implementation notes tied to a concrete project.
- `reading-notes`: paper or book chapter notes.
- `research-notes`: early research questions, hypotheses, and reflections.
